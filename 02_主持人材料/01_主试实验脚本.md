# 主试实验脚本

## 0. 实验前准备

主试提前准备：

- 问卷二维码
- 对应被试的任务文件夹，例如 `03_被试任务卡/P1`
- RefineUI 系统
- GPT Baseline 页面
- Figma
- 计时器

开始前确认被试编号，例如 P1、P2、P3。请让被试只打开自己编号对应的文件夹。

## 1. 开场说明

你好，感谢你参加本次实验。本实验主要了解不同 UI 评估辅助方式对界面修改的帮助。

这不是测试你的设计能力。你只需要按照文件夹里的说明，使用指定系统查看建议，并在 Figma 中修改 UI。

每次正式实验的修改时间为 10 分钟。时间到了以后，请停止当前修改，保存文件，然后进入下一次实验。

实验过程中请不要关闭问卷页面，后面每次实验结束后都要继续填写对应页面。

## 2. 基础信息与练习

请先扫描问卷二维码，填写第 1 页基础信息。填写完成后不要关闭问卷页面。

然后打开你的被试文件夹，先进入 `实验练习` 文件夹，阅读里面的 `README.txt`，完成两个系统的练习：

- RefineUI 练习
- GPT Baseline 练习

练习只用于熟悉流程，不计入正式结果。主试可以在练习阶段帮助被试理解如何上传截图、上传 JSON、使用提示词、查看输出和保存 Figma 文件。

## 3. 正式实验通用流程

被试需要按顺序完成：

1. 第一次实验
2. 第二次实验
3. 第三次实验
4. 第四次实验

每次实验开始时，主试提示：

现在进行第 X 次实验。请打开对应实验文件夹里的 `README.txt`，按里面的步骤操作。本次修改时间为 10 分钟，我会帮你计时。

主试在被试开始修改 Figma 时开始 10 分钟倒计时。时间到后提示：

时间到了，请停止修改，保存当前修改稿。请不要关闭问卷页面，接下来填写本次实验对应的问卷页，然后进入下一次实验。

## 4. 主试检查内容

每次实验结束后，主试只需要检查文件是否保存到对应位置，必要时帮助被试保存。

检查内容：

- 系统输出是否保存到本次实验文件夹中对应的 `系统输出.txt` 文件。
- 参与者修改稿是否保存为本次实验文件夹中对应的 `参与者修改稿.fig` 文件。
- G组实验是否使用了该实验文件夹中的提示词文件。
- R组实验没有提示词文件，不需要检查提示词。
- 被试是否填写了本次实验对应的问卷页面。
- 问卷页面不要关闭，后续继续填写。

不需要录屏。

## 5. RefineUI 实验提示

如果本次 README 写的是 RefineUI，主试只提醒被试：

- 打开 RefineUI。
- 上传本次实验文件夹中的截图。
- 上传或粘贴本次实验文件夹中的 JSON 文件。
- 读取系统输出。
- 根据建议修改 Figma 文件。

## 6. GPT Baseline 实验提示

如果本次 README 写的是 GPT Baseline，主试只提醒被试：

- 新建一次全新的 GPT 对话。
- 上传本次实验文件夹中的截图。
- 输入或粘贴本次实验文件夹中的 JSON 文件。
- 使用本次实验文件夹中的提示词。
- 只生成一次回答，不继续追问。
- 根据这一次回答修改 Figma 文件。

## 7. 四次实验完成后

四次正式实验全部完成后，主试提示：

请继续填写问卷第 6 页整体评估。填写完成后不要立刻离开，接下来会进行一个简短访谈。

## 8. 访谈

问卷完成后，主试按照 `03_访谈大纲.md` 进行简短访谈。

访谈结果保存到 `04_访谈结果` 中对应被试文件夹。

## 9. 结束语

感谢你的参与。本次实验已经结束。后续数据会匿名整理，只用于研究分析。
